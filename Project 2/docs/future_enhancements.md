# Future Enhancements

This section outlines improvements that make the project stronger for a portfolio or production showcase.

## Suggested Improvements

- Add a natural-language command parser so users can type phrases instead of menu choices.
- Persist experiment results to JSON, CSV, or a lightweight database.
- Add cross-validation and learning curves for model stability analysis.
- Add a visual report generator for confusion matrices and metric comparisons.
- Implement model saving and loading for reuse without retraining.
- Add more dataset support beyond Iris, such as Wine or Digits.
- Add a web API or Flask/Streamlit UI for interactive use.
- Add a detailed experiment logging dashboard.
- Add more advanced algorithms such as SVM or RandomForest.

## Portfolio Enhancements

- Document model explainability with feature importance.
- Add code comments and architecture rationale to improve readability.
- Add CI/CD configuration for tests and linting.
- Add a `requirements-dev.txt` with linters and formatters.
- Add reproducible environment configuration via `pyproject.toml`.
